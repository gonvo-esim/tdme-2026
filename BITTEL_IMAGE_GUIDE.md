## General Image Specifications

### File Format & Compression
- **Format**: WebP (primary), fallback to PNG for legacy support
- **Color Space**: sRGB (for web consistency)
- **Compression Level**: Optimized for web (85-90% quality)
- **File Size**: < 150KB for grid images, < 300KB for hero/showcase sections

### Naming Convention
- **Pattern**: `{product-name}-{type}.webp`
- **Examples**: `unomedia-5-series-main.webp`, `neo-dect-alt.webp`
- **Types**: `main` (primary product image), `alt` (alternative angle/detail)
- **Use lowercase** with hyphens only (no spaces or underscores)

## Responsive Image Behavior

### Grid Display (Product Categories)
- **Desktop** (≥1024px): 800×600px
- **Tablet** (768-1023px): 600×450px
- **Mobile** (<768px): 400×300px

### Hero Sections
- **Desktop**: 1920×600px
- **Tablet**: 1280×400px
- **Mobile**: 640×200px

### Fallback & Placeholder
- **Placeholder** (`placeholder_bittel_all.webp`): 400×300px, neutral gray background
- Used when specific product images are unavailable

## Image Optimization Checklist

- [ ] All images converted to WebP format
- [ ] File size optimized (< 150KB for grid, < 300KB for showcase)
- [ ] Color space: sRGB
- [ ] Dimensions match category specifications
- [ ] Aspect ratio correct (4:3 for products, 16:5 for hero)
- [ ] Proper alt text for accessibility
- [ ] No watermarks or branding elements (unless specified)
- [ ] Consistent lighting across category
- [ ] Professional background (white, gray, or neutral)
- [ ] Product clearly visible and sharp focus
- [ ] Filename follows naming convention
- [ ] File stored in correct category subdirectory

## Implementation Notes

1. **Progressive Enhancement**: Provide responsive image sets using `srcset` for different screen sizes
2. **Lazy Loading**: Implement lazy loading for below-the-fold images
3. **Accessibility**: Every image must have descriptive alt text following the pattern provided
4. **Consistency**: Maintain consistent lighting, background, and positioning across categories
5. **Performance**: Monitor image load times and optimize if needed
6. **Updates**: When updating product images, maintain the same filename and dimensions to avoid breaking links
